# CSX LSP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a WebSocket LSP endpoint for UniEmu `.csx` scripts with Roslyn-backed diagnostics and completions that understand DB-backed `#load` scripts.

**Architecture:** The backend exposes `/csx-lsp` and hosts an OmniSharp.Extensions.LanguageServer LSP server per WebSocket connection. Custom handlers keep unsaved document overlays, resolve `#load` against open documents and `ScriptFiles`, and call a focused Roslyn language service shared by diagnostics/completion tests. The frontend connects Monaco to the LSP and keeps the existing local snippets as fallback.

**Tech Stack:** ASP.NET Core WebSockets, OmniSharp.Extensions.LanguageServer, Nerdbank.Streams, Roslyn scripting/compilation APIs, Monaco editor, monaco-languageclient.
