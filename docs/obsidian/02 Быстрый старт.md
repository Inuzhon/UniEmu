# Быстрый старт

## Требования

Backend использует .NET 10 preview. Frontend использует React/Vite и зафиксирован на `yarn@1.22.22`. Основная БД - SQLite.

Нужны:

- .NET SDK 10 preview;
- Node.js, совместимый с текущими frontend-зависимостями;
- Yarn 1.x;
- Docker, если нужен контейнерный запуск.

## Локальный запуск backend

Из корня репозитория:

```powershell
dotnet run --project UniEmu/UniEmu.csproj
```

По умолчанию backend слушает порт `5083`. Порт задается настройкой `UniEmu:Port`.

Development OpenAPI доступен только в development-окружении через встроенный `MapOpenApi()`.

## Локальный запуск frontend

Из каталога `UniEmu.Client`:

```powershell
yarn install
yarn dev
```

Vite dev server слушает `http://localhost:8070`.

В dev-режиме Vite проксирует:

- `/api` на `http://localhost:5083`;
- `/hubs` на `http://localhost:5083` с WebSocket.

Цель прокси можно изменить переменной `VITE_API_PROXY_TARGET`.

## Docker Compose

Из корня репозитория:

```powershell
docker compose up --build
```

Compose поднимает сервис `uniemu`, публикует порт `${UNIEMU_PORT:-5083}:5083`, сохраняет данные в volume `uniemu-data`, логи в `uniemu-logs`.

Важные переменные:

| Переменная | Назначение |
| --- | --- |
| `UNIEMU_PORT` | Внешний порт контейнера. |
| `UNIEMU_TIME_ZONE` | Часовой пояс приложения, по умолчанию `Europe/Moscow`. |
| `UNIEMU_DB_CONNECTION` | SQLite connection string внутри контейнера. |
| `ASPNETCORE_ENVIRONMENT` | ASP.NET Core environment. |

## Проверка проекта

Backend-тесты:

```powershell
$env:DOTNET_CLI_USE_MSBUILD_SERVER='0'
dotnet test UniEmu.Tests/UniEmu.Tests.csproj --no-restore -nr:false -p:UseSharedCompilation=false -p:SkipYarnBuild=True
```

Frontend-команды:

```powershell
yarn lint
yarn build
```

> [!NOTE]
> В существующих агентских заметках указано, что `dotnet build` и `dotnet test` иногда зависают из-за build servers. Если CLI-запуск подвисает, сначала выполните `dotnet build-server shutdown`.

## Первые действия в UI

1. Откройте `http://localhost:8070` при отдельном frontend-запуске или backend origin при production/static hosting.
2. Перейдите в раздел эмуляторов.
3. Создайте эмулятор с именем, `ProtocolId`, `TargetUrl` и интервалом отправки.
4. Добавьте теги: статические, генераторные, сценарные или скриптовые.
5. Запустите эмулятор.
6. Откройте карточку эмулятора и наблюдайте telemetry/realtime-обновления.
