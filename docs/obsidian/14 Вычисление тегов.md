# Вычисление тегов

Эта страница описывает теги с точки зрения пользователя: какие способы получения значения есть, чем они отличаются и когда какой выбирать.

## Общая модель

Каждый тег в UniEmu отвечает на три вопроса:

1. Что отправлять: `Key`, `Name`, `Type`.
2. Откуда брать значение: `Source`.
3. Когда пересчитывать: `Trigger`.

Runtime в момент публикации собирает все теги эмулятора, приводит значения к типу тега и отправляет включенные (`Enabled`) теги в Dispatcher.

## Типы значений

| Тип | Что означает | Как приводится значение |
| --- | --- | --- |
| `int` | Целое число. | Число округляется до `int`. |
| `double` | Число с плавающей точкой. | Может округляться через `RoundDigits`. |
| `string` | Строка. | Для static/script/cnc берется текстовое значение, для генератора число превращается в строку. |
| `bool` | Логическое значение. | `0` становится `false`, любое ненулевое значение становится `true`. |

## Источники значения

### Static

`static` - самый простой источник. Значение берется из `Preview`.

Когда использовать:

- постоянный флаг;
- имя программы;
- фиксированный текст;
- параметр, который меняется только вручную или скриптом.

Особенность: CSX-скрипт может менять static-теги через `UniEmu.Tags.TrySetValue(...)`.

Для static string-тегов со специальными параметрами `PrgName` и `Subprogram` UI помогает выбрать значение из хранилища CNC-программ. Вместо ручного ввода показывается выпадающий список с поиском и группами `Общие УП` / `УП этого эмулятора`. В сам тег сохраняется только имя программы, поэтому runtime по-прежнему работает со строкой из `Preview`.

### Generator

`generator` рассчитывает значение по формуле от времени, прошедшего после запуска эмулятора.

Когда использовать:

- плавная нагрузка;
- имитация температуры;
- синусоидальные колебания;
- простой шум;
- повторяющийся сигнал.

Все функции генератора описаны ниже в разделе [[14 Вычисление тегов#Функции генератора]].

### Scenario

`scenario` - цепочка нескольких generator-сегментов. Каждый сегмент имеет собственную формулу и длительность.

Когда использовать:

- разгон, рабочий режим и останов;
- цикл обработки детали;
- прогрев, стабилизация и авария;
- длинная демонстрация с несколькими фазами.

Подробно: [[15 Сценарии тегов]].

### Script

`script` рассчитывает значение через `.csx`-скрипт.

Когда использовать:

- нужна ветвистая логика;
- значение зависит от других тегов;
- требуется persistent state;
- нужно обратиться к разрешенной REST-операции;
- генератора и сценария недостаточно.

Подробно: [[16 Редактор скриптов и API]].

### Formula

`formula` тоже использует CSX-скрипт. В текущей реализации это скриптовый источник без предварительно рассчитанного generator-значения. Его удобно воспринимать как "скриптовая формула для тега".

### FormulaScript

`formulaScript` - комбинированный режим:

1. Runtime сначала считает generator-значение по `Calc`.
2. Затем передает это значение в CSX-скрипт как текущее.
3. Скрипт может модифицировать результат.

Когда использовать:

- базовый сигнал строится формулой, а скрипт добавляет условия;
- нужно ограничить диапазон;
- нужно учитывать другой тег;
- нужно добавить внешний фактор через `UniEmu.Rest`.

### CNC

`cnc` обозначает значение, связанное с контекстом управляющей программы. Наиболее важные CNC-сценарии сейчас реализованы через специальные параметры: `PrgName`, `Subprogram`, `FrameNum`, `FrameText`.

Связка параметров работает так:

1. `PrgName` содержит имя основной CNC-программы.
2. `Subprogram` содержит имя подпрограммы и, если она найдена, имеет приоритет для расчета кадров.
3. `FrameNum` вычисляется runtime по текущему времени с момента запуска эмулятора.
4. `FrameText` получает строку CNC-программы с тем же номером кадра.

Для программы из трех строк при `IntervalSec = 1` кадры будут идти так: `0`, `1`, `2`, затем снова `0`, `1`, `2`. Если программа не найдена или пуста, runtime публикует `FrameNum = 0` и пустой `FrameText`.

## Триггеры вычисления

| Trigger | Что делает | Пример |
| --- | --- | --- |
| `once` + `onStart` | Считает один раз при запуске эмулятора. | Начальное состояние, сброс счетчика. |
| `once` + `onStop` | Считает один раз при остановке. | Финальный статус, отметка остановки. |
| `interval` | Считает каждые N ms/sec/min. | Температура каждые 500 ms. |
| `cron` | Считает по cron-выражению. | Смена статуса каждый час. |

Главный publish interval эмулятора управляет частотой отправки всего пакета. Отдельный trigger тега управляет частотой пересчета конкретного тега.

## Функции генератора

Генератор получает elapsed time от запуска эмулятора. Большинство функций используют:

- `start` - начальное значение;
- `finish` - конечное значение;
- `duration` - длительность одного прохода;
- `amplitude` - амплитуда периодического сигнала;
- `period` - период периодического сигнала;
- `curvature` - степень кривизны;
- `distortion` - процент искажения.

### None

Функция генератора не применяется. Runtime использует `Preview`.

Подходит, когда значение должно оставаться начальным, но интерфейс требует calc config.

### Text

Текстовое значение. В runtime числовой генератор фактически не меняет значение, для строковых тегов используется preview-текст.

Подходит для статичных текстов, но для обычного текста чаще проще выбрать `static`.

### Line

Линейный переход от `start` к `finish` за `duration`.

Формула по смыслу:

```text
value = start + (finish - start) * progress
```

Пример: нагрузка шпинделя растет с `10` до `75` за `30` секунд.

### Curve

Нелинейный переход от `start` к `finish`. Поле `curvature` задает степень кривой.

Если `curvature > 1`, изменение медленнее в начале и быстрее ближе к концу. Если значение близко к `1`, поведение похоже на `Line`.

Пример: температура сначала растет медленно, затем быстрее.

### Sequence

Выбор значения из JSON-массива, указанного в `start`.

Пример `start`:

```json
[0, 10, 20, 40, 80]
```

Runtime выбирает элемент по прогрессу внутри duration. Это удобно для ступенчатых режимов, кодов состояния или дискретных уровней.

> [!NOTE]
> Backend поддерживает `Sequence`, но текущий frontend type list местами ограничивает выбор calc types. Если в UI нет этой функции, это ограничение интерфейса, а не генератора backend.

### Random

Случайное значение между `start` и `finish`.

Пример: имитация небольшого хаоса датчика, если не нужна физически точная модель.

Для более мягкого шума можно использовать `Line` или `Sinusoid` и добавить `distortion`.

### Sinusoid

Синусоидальный сигнал:

```text
value = start + amplitude * sin(2π * elapsed / period)
```

Поля:

- `start` - центр колебания;
- `amplitude` - отклонение от центра;
- `period` - длительность полного цикла.

Пример: нагрузка колеблется около `50` с амплитудой `10`.

### Square

Прямоугольный сигнал. Значение переключается между `start + amplitude` и `start - amplitude`.

Пример: датчик "цикл активен" или имитация включения/выключения.

### Sawtooth

Пилообразный сигнал. Значение постепенно растет или падает внутри периода и затем сбрасывается.

Пример: счетчик фазы, циклическая координата, условная загрузка буфера.

### SquircleEarly

Сглаженная кривая, которая быстрее набирает значение в начале и мягче подходит к концу.

Подходит для разгона, когда основной рост должен случиться рано.

### SquircleLate

Сглаженная кривая, которая медленнее стартует и активнее меняется ближе к концу.

Подходит для запаздывающего процесса: прогрев, накопление давления, выход на режим.

## Distortion

`distortion` задается в процентах. Runtime добавляет псевдошум, масштабированный относительно текущего значения.

Когда полезно:

- сделать график менее идеальным;
- имитировать дрожание датчика;
- показать живую телеметрию вместо ровной математической линии.

Когда не стоит:

- для кодов состояния;
- для bool-тегов;
- для точных идентификаторов;
- для имени программы и текстовых тегов.

## Округление

`RoundDigits` применяется только к `double`. Значение ограничивается диапазоном 0..15. Округление выполняется `AwayFromZero`.

Пример:

| Исходное | `RoundDigits` | Результат |
| --- | --- | --- |
| `12.3456` | `2` | `12.35` |
| `12.3456` | `0` | `12` |

## Практический выбор источника

| Нужно | Выберите |
| --- | --- |
| Постоянное значение | `static` |
| Простое плавное изменение | `generator` + `Line` |
| Периодика | `generator` + `Sinusoid`, `Square` или `Sawtooth` |
| Несколько фаз подряд | `scenario` |
| Логика с условиями | `script` или `formula` |
| Формула плюс скриптовая корректировка | `formulaScript` |
| Работа с именем программы и кадрами | `static/scenario` + `SpecialParameter` |
