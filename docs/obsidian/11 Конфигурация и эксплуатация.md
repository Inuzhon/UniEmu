# Конфигурация и эксплуатация

## Backend settings

Основная секция:

```json
{
  "UniEmu": {
    "Port": 5083,
    "TimeZone": "Europe/Moscow",
    "Culture": "ru-RU",
    "DispatcherBlockCheckIntervalSeconds": 5,
    "EnableStaticAssetCompression": true,
    "EnableStaticAssetCaching": true
  }
}
```

Важные опции:

| Опция | Назначение |
| --- | --- |
| `Port` | HTTP-порт backend. |
| `TimeZone` | Часовой пояс приложения. |
| `Culture` | Культура форматирования. |
| `DisableRuntime` | Отключает Quartz hosted service. |
| `SkipStartupDatabase` | Пропускает startup database initialization. |
| `SeedData` | Управляет seed-данными. |
| `DisableStaticAssets` | Отключает отдачу frontend static assets. |
| `EnableStaticAssetCompression` | Включает response compression для static assets. |
| `EnableStaticAssetCaching` | Включает cache headers для production assets. |
| `DispatcherBlockCheckIntervalSeconds` | Интервал проверки блокировки Dispatcher, runtime clamp 5..10 секунд. |

## Connection string

```json
{
  "ConnectionStrings": {
    "UniEmuDb": "Data Source=uniemu.db"
  }
}
```

В Docker Compose значение обычно:

```text
Data Source=/app/data/uniemu.db
```

## RestCatalog

Секция `UniEmu:RestCatalog` описывает разрешенные REST-операции для CSX-скриптов. Это не общий HTTP-доступ, а белый список операций.

Пример:

```json
{
  "DefaultClient": {
    "BaseUrl": "https://external-system.local",
    "TimeoutSeconds": 5,
    "Headers": {},
    "Operations": {
      "GetWorkerById": {
        "Method": "GET",
        "Path": "/api/workers/{workerId}",
        "Response": "Worker"
      }
    }
  }
}
```

## Логи

Serilog пишет:

- в консоль;
- в rolling files `Logs/uniemu-.log`.

По умолчанию хранятся 14 файлов, лимит файла 10 MB, включен roll on size limit.

В Docker logs volume:

```text
uniemu-logs:/app/Logs
```

## Static frontend assets

Backend может отдавать production frontend assets и fallback. В dev frontend обычно запускается отдельно на `8070`, а API проксируется в backend.

В production:

- frontend assets отдаются ASP.NET Core;
- API доступен по тому же origin через `/api`;
- SignalR через `/hubs/runtime-updates`.

PWA service worker и manifest также живут на этом frontend origin. Подробности: [[17 PWA и офлайн-режим]].

## Frontend env

| Переменная | Назначение |
| --- | --- |
| `VITE_API_BASE_URL` | Явный backend origin. |
| `VITE_API_PROXY_TARGET` | Цель dev proxy для `/api` и `/hubs`. |
| `VITE_PERSIST_STORE` | Включает localStorage persist для части store. |

## Docker Compose

Compose использует:

- image `${UNIEMU_IMAGE:-uniemu:local}`;
- container name `${UNIEMU_CONTAINER_NAME:-uniemu}`;
- restart `unless-stopped`;
- порт `${UNIEMU_PORT:-5083}:5083`;
- SQLite volume;
- logs volume.

## Проверки и диагностика

Backend:

```powershell
$env:DOTNET_CLI_USE_MSBUILD_SERVER='0'
dotnet test UniEmu.Tests/UniEmu.Tests.csproj --no-restore -nr:false -p:UseSharedCompilation=false -p:SkipYarnBuild=True
```

Frontend:

```powershell
yarn lint
yarn build
```

Если `dotnet`-команды зависают, сначала:

```powershell
dotnet build-server shutdown
```

## Эксплуатационные замечания

- Аутентификация сейчас не включена, поэтому не стоит выставлять UI/API в недоверенную сеть без внешней защиты.
- Dispatcher HTTP client принимает любые SSL-сертификаты.
- Runtime state частично in-memory, поэтому аварийное завершение может потерять последние preview-значения.
- SQLite подходит для локального и небольшого стендового сценария, но для распределенной эксплуатации потребуется отдельная архитектурная оценка.
